USAGE EXAMPLES
================================================================================

QUICK START (5 Minutes)
────────────────────────────────────────────────────────────────────────────
1. Create language:
   parsercraft create --preset python_like --output lang.yaml

2. Generate VS Code extension:
   parsercraft extension --config lang.yaml --output ext

3. Install:
   cd ext && npm install && npm run compile
   code --install-extension .

4. Result: Full IDE support! ✓

MODULAR PROJECT (Type Check + Modules)
────────────────────────────────────────────────────────────────────────────
1. Create modules with exports
2. Import in main file
3. Check dependencies:
   parsercraft module-deps main
4. Verify no cycles:
   parsercraft module-cycles
5. Type check:
   parsercraft type-check --input file.teach --level strict

CI/CD PIPELINE
────────────────────────────────────────────────────────────────────────────
1. Validate config:
   parsercraft validate lang.yaml
2. Check cycles:
   parsercraft module-cycles
3. Type check all files:
   for f in **/*.teach; do
     parsercraft type-check -c lang.yaml -i $f -l strict
   done
4. Run tests:
   parsercraft test --config lang.yaml --tests tests.yaml
5. Generate deployment:
   parsercraft extension --config lang.yaml

DATA STRUCTURES
────────────────────────────────────────────────────────────────────────────
Type System:
  - Type: Represents types with kind, name, generics, nullability
  - TypeEnvironment: Scope management with parent chains
  - TypeSignature: Function signatures with parameters and return type
  - ClassType: User-defined types

Module System:
  - Module: Represents single module with metadata
  - ModuleImport: Represents imported name with version constraints
  - ModuleExport: Represents exported name with visibility
  - ModuleMetadata: Package metadata from module.yaml
